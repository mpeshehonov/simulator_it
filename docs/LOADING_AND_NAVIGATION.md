# Загрузка и навигация: как избежать дёрганий и пустых экранов

Кратко структурированные подходы, чтобы не показывать пустой контент и не мигать при переходах.

---

## 1. Когда показывать экран загрузки

- **Показывать loading только когда нет данных для отображения.**  
  Если после навигации в сторе уже есть нужные данные (например, игрок после онбординга), не показывать лоадер — отображать контент сразу. Рефетч в фоне делать без переключения в состояние «загрузка».

- **Условие в коде:**  
  `showLoading = initData && isLoading && !needsOnboarding && !player`  
  То есть: загрузка только когда мы ещё не знаем, есть ли игрок (первый заход или повторный запрос без кэша).

---

## 2. Не ставить loading при фоновом рефетче

- В хуке загрузки (например, `usePlayer`) **не вызывать `setLoading(true)`**, если в сторе уже есть сущность (например, `player`).  
  В этом случае делать тихий рефетч и по завершении обновить стор; экран при этом остаётся с текущим контентом.

- Итог: после перехода с онбординга на главную не мигает «пустой экран → лоадер → контент».

---

## 3. Один источник правды и фазы приложения

- Состояние (игрок, needsOnboarding, phase) хранить в одном месте (например, Zustand).  
  Редиректы и решение «что рендерить» делать по этому состоянию, а не по локальному `useState` в странице.

- Имеет смысл ввести фазы: `idle` → `auth` → `onboarding` | `ready` → `error`.  
  Тогда на главной: «показывать лоадер» только когда фаза ещё не `ready` и данных для экрана нет.

---

## 4. Скелетоны вместо «пустого места»

- Если контент грузится частями, лучше показывать скелетон той же формы, что и финальный блок (карточки, списки), а не пустой белый экран. Так меньше визуальных скачков и ощущение «дёргания».

---

## 5. Next.js и Suspense (если понадобится)

- В App Router можно использовать `loading.js` и React Suspense: лоадер показывается на уровне сегмента роута до готовности данных.  
  Для клиентского стора (как у нас) важнее правило из п. 1–2: не переключать в loading, если данные уже есть.

---

## 6. Чего избегать

- Не вызывать `setLoading(true)` при каждом монтировании страницы, если данные уже в сторе.  
- Не редиректить на главную и одновременно сбрасывать игрока в сторе — иначе на главной снова «нет данных» и покажется лоадер или пустой экран.  
- Не рендерить финальный layout с пустыми блоками («пустой главный экран») на один кадр перед показом лоадера — условие показа лоадера должно учитывать наличие данных (см. п. 1).
